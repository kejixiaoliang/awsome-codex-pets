# License Policy

Codex pets are creative assets. Treat every pet as copyrighted unless the source
clearly says otherwise.

## Mirror Rules

Only mirror `pet.json`, `spritesheet.webp`, or package archives when one of the
following is true:

- The source declares a license that allows redistribution.
- The author explicitly says mirroring or redistribution is allowed.
- The asset is in a repository with a license that covers the pet files.

If the license is unclear, keep the pet as `link-only`.

License-unclear files may be downloaded into `downloads/review/` for local
inspection. That directory is ignored by git and must not be published until
redistribution rights are confirmed.

## Attribution

Every catalog entry should preserve:

- Original source URL
- Author or publisher name when available
- License text or license URL when available
- Date the metadata was fetched

## Safety

- Do not run third-party install scripts just to catalog a pet.
- Prefer direct static downloads.
- Record SHA-256 hashes for mirrored files.
- Keep generated or transformed files separate from original assets.
